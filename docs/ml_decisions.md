# Осознанность выбора ML-подходов

## Эмбеддинги: cointegrated/rubert-tiny2

**Почему выбрали:**
- Компактная модель (312-dim вместо 768), в 2× быстрее на inference
- Обучена на русскоязычных текстах — понимает морфологию и контекст
- Помещается в RAM за ~100MB, не требует GPU для базовой нагрузки

**От чего отказались:**
- `sbert-large` — слишком тяжёлая для embedded-режима, латентность >200ms/пост
- TF-IDF + cosine — нет семантического понимания: "мошенник" и "аферист" дают нулевое сходство
- Word2Vec / FastText — статические эмбеддинги, не учитывают контекст предложения
- OpenAI Embeddings — внешний API, задержка сети, стоимость, недоступность в RU

**Цена компромисса:** 312-dim вместо 768 даёт чуть хуже разделение семантически близких кластеров, но на нашем домене (финансовые угрозы) разница незначительна.

## Дедупликация: SimHash + LSH-бакеты в Redis

**Почему выбрали:**
- SimHash даёт 64-битный fingerprint за O(n) по числу токенов
- LSH-бакеты позволяют искать near-duplicates без попарного сравнения — O(1) lookup
- Redis с TTL 24h — автоматическая очистка без ручного управления

**От чего отказались:**
- MinHash + MinHashLSH — хорош для jaccard similarity, но медленнее на коротких текстах
- Exact hash (MD5) — ловит только точные дубли, не near-duplicates (репосты с правкой)
- Embedding cosine dedup — дорого (~40ms/пост), нет смысла гонять модель дважды

**Граница применимости:** порог Hamming ≤ 3 из 64 бит. Полностью переписанный дубликат (те же факты, другие слова) не обнаруживается.

## Кластеризация: Qdrant (ANN) + Redis metadata

**Почему выбрали:**
- Qdrant даёт приближённый поиск ближайших соседей за O(log n) — масштабируется до млн векторов
- Cosine distance соответствует нашей задаче (нормализованные эмбеддинги)
- Redis для metadata — мгновенный доступ к cluster_id без запроса к Qdrant

**От чего отказались:**
- FAISS — нет встроенного HTTP API, сложнее деплоить в микросервисной архитектуре
- K-Means — требует предопределённого числа кластеров и переобучения при каждом новом посте
- DBSCAN — не работает онлайн (нужны все данные сразу), не подходит для streaming pipeline

**Порог similarity 0.82:** эмпирически подобран на тестовом датасете. Ниже 0.82 — кластеры размываются несвязанными постами, выше 0.82 — тематически близкие посты попадают в разные кластеры.

## Фильтрация: pymorphy3 + substring fallback

**Почему выбрали:**
- Морфологический анализатор для русского языка — "мошенников" → "мошенник"
- Без него keyword "мошен" не матчит "мошенничество" в лемматизированном тексте
- Substring fallback как страховка для частичных стемов

**От чего отказались:**
- Snowball stemmer — стемминг грубее лемматизации, больше false positives
- Чистый substring match — регистр, словоформы, падежи не обрабатываются
- ML-классификатор релевантности — избыточно для первичной фильтрации по ключевым словам
