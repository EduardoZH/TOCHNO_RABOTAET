# Контур обновления и версионирования модели

## Текущее состояние

Используется претренированная модель `cointegrated/rubert-tiny2` без дообучения.
Версия фиксируется через env-переменную `EMBEDDING_MODEL_VERSION` (default: `1.0.0`) и логируется при каждом старте embedding-сервиса.

## Как обновить модель

1. **Подготовить данные** — размеченный датасет постов (релевант/нерелевант + тональность)
2. **Дообучить** — fine-tune rubert-tiny2 на доменных данных (скрипт `scripts/finetune.py`, не реализован в текущей версии)
3. **Сохранить веса** в `models/rubert-tiny2-v2.0/`
4. **Обновить переменную** `EMBEDDING_MODEL=./models/rubert-tiny2-v2.0` в `docker-compose.yml`
5. **Обновить версию** `EMBEDDING_MODEL_VERSION=2.0.0`
6. **Пересобрать образ** `docker-compose build embedding`
7. **Накатить rolling update** — поднять новый контейнер embedding, старый продолжает работать пока не завершит текущие сообщения

## Связь версий

| Компонент | Переменная | Пример |
|-----------|-----------|--------|
| Embedding-модель | `EMBEDDING_MODEL` | `cointegrated/rubert-tiny2` |
| Версия модели | `EMBEDDING_MODEL_VERSION` | `1.0.0` |
| Порог кластеризации | `SIMILARITY_THRESHOLD` | `0.82` |
| Порог дедупликации | `DEDUP_HAMMING_THRESHOLD` | `3` |

Все параметры хранятся в `docker-compose.yml` → воспроизводимость гарантирована.

## Деградация качества

При смене модели нужно пересчитать все эмбеддинги в Qdrant — старые векторы несовместимы с новой моделью.
Для этого: очистить коллекцию Qdrant (`DELETE /collections/posts`) и переиндексировать данные.
